# Landly — 사용자 관점 개선 포인트

> 작성일: 2026-04-30  
> 버전: v44 기준 분석  
> 대상: 한국 방문 외국인 (여행자 + 정착자) 사용자 경험

---

## 🔴 Critical — 실사용 진입 장벽

### 1. 100% Mock 데이터 — 실제 정보 없음

- 장소, 병원, 약국, 교통 데이터가 전부 가상 정보
- 사용자가 실제로 찾아가면 틀린 정보일 가능성 높음
- **개선안:** 공공데이터 API 연동
  - 한국관광공사 TourAPI (장소, 관광지)
  - 카카오맵 / 네이버 지도 API (실시간 위치, 거리)
  - 서울시 공공데이터 포털 (병원, 약국)

---

### 2. 언어 지원 불균형

| 언어 | UI 번역 | 프레이즈 카드 |
|---|---|---|
| 한국어 (ko) | ✅ 완전 지원 | ✅ |
| 영어 (en) | ✅ 완전 지원 | ✅ |
| 중국어 (zh) | ❌ 영어 UI 그대로 | ✅ |
| 일본어 (ja) | ❌ 영어 UI 그대로 | ✅ |
| 스페인어 (es) | ❌ 영어 UI 그대로 | ✅ |
| 프랑스어 (fr) | ❌ 영어 UI 그대로 | ✅ |

- 일본어·중국어 사용자가 메뉴 자체를 읽을 수 없어 이탈 확실
- **개선안:** `i18n/zh.json`, `i18n/ja.json` 등에 UI 전체 번역 키 추가

---

### 3. 오프라인 캐시 범위가 너무 좁음

현재 캐시 대상: `/`, `/onboarding`, `/pass`, `/shop`, `/care`, `/sos`

- `/care` 진입은 되지만 `/care` 하위 기능(병원 목록 등) 오프라인 불가
- 긴급상황에서 정작 필요한 상세 페이지 접근 불가
- **개선안:** 핵심 하위 경로 및 SOS 관련 데이터 전체를 서비스 워커 캐시에 포함

---

## 🟠 High — UX 마찰

### 4. 온보딩 5단계 — 이탈 유발

- 앱 첫 실행 시 5단계 위저드를 강제 통과해야 함
- "그냥 써보고 싶다"는 사용자 다수 이탈
- **개선안:**
  - 필수 2단계(언어 + 방문 목적)만 유지
  - 나머지 3단계는 프로필(`/my`)에서 나중에 설정 가능하도록 이동
  - "Skip for now" 버튼 추가

---

### 5. 하단 내비게이션 구조 문제

- 5개 탭 중 마지막이 "More" → 주요 기능이 숨겨짐
- Care, Shop, Stay 같은 핵심 기능이 More 아래에 묻히는 구조
- **개선안:** 사용 모드(여행자 / 정착자)에 따라 하단 탭 구성을 동적으로 변경
  - 여행자: Home / Pass / Explore / Care / SOS
  - 정착자: Home / Stay / Life / Care / More

---

### 6. 글로벌 검색 없음

- 프레이즈, 장소, 병원, SOS 시나리오를 각 탭에서 개별 검색만 가능
- 긴급상황에서 원하는 정보를 빠르게 찾기 어려움
- **개선안:** 홈 상단에 통합 검색 진입점 추가 (프레이즈 + 장소 + 병원 + SOS 통합 인덱스)

---

### 7. 지도 연동이 "핸드오프"에 불과

- 장소 선택 → 외부 앱 열기로만 끝남
- 앱 내에서 위치 맥락 전혀 없음 (내 주변, 거리 표시 없음)
- **개선안:** 카카오맵 Static Map API로 장소 미리보기 지도 표시 (인터랙션 없이도 위치 파악 가능)

---

### 8. 의료 트리아지 결과가 지나치게 단순

- 증상 입력 → "병원 가세요" 수준의 추천
- 실제 필요 정보: "가까운 영어 가능 병원, 몇 호선 몇 분 거리"
- **개선안:** 트리아지 결과에 실제 병원 데이터 연동 + 교통 정보 연결

---

## 🟡 Medium — 개선 시 체류시간/신뢰도 향상

### 9. 환급(Tax Refund) 플로우 — 실제 계산 불가

- RefundCheckerForm이 환급 "가능 여부"만 판단, 실제 금액 계산 없음
- 영수증 사진 업로드 UI는 있으나 실제 처리 없음 (저장만 됨)
- **개선안:** 환급 금액 계산 로직 구현 (구매 금액 × 환급율 공식 적용)

---

### 10. 베타 테스트 기능이 프로덕션에 노출

- `/test` 페이지, BetaMissionCard, PilotQaChecklist가 일반 사용자에게도 보임
- `/more` 탭에 "Beta Tester" 진입 버튼 노출
- **개선안:** 환경변수 feature flag 또는 별도 빌드로 베타 기능 분리

---

### 11. 알림/리마인더 — 실제 푸시 없음

- 리마인더 설정 가능하지만 PWA Push Notification 미연동
- 사용자가 앱 밖에 있을 때 알림 수신 불가
- **개선안:** Web Push API 연동 (서비스 워커 기반 푸시 알림 구현)

---

### 12. 캘린더 — 외부 연동 없음

- 이벤트/마감일이 앱 캘린더에만 저장됨
- 구글/애플 캘린더 export 없음 → 실사용에서 분리됨
- **개선안:** `.ics` 파일 export 기능 추가 (Google Calendar, Apple Calendar 호환)

---

### 13. 저장 아이템 백업/공유 불가

- 저장한 장소, 프레이즈, 리마인더가 localStorage에만 있음
- 폰 교체 또는 캐시 삭제 시 모두 사라짐
- **개선안:** JSON export/import 기능 추가 (최소 백업 용도)

---

## 🟢 Low — 완성도 Polish

### 14. 다크모드 미지원

- 시스템 다크모드를 따르지 않음
- 야간 야외 사용 시 눈 부심
- **개선안:** Tailwind `dark:` 클래스 적용 + `prefers-color-scheme` 미디어 쿼리 대응

---

### 15. 파트너 오퍼 신뢰도 문제

- 파트너 오퍼가 100% mock 데이터 + "Commercial Disclosure" 배너
- 실제 딜이 없는 상태에서 파트너 탭이 메인에 노출되면 신뢰도 하락
- **개선안:** 실제 파트너 계약 전까지 파트너 탭 숨김 또는 "Coming Soon" 처리

---

### 16. SOS 스크립트 — 발음 정보 없음

- 긴급상황 한국어 스크립트가 텍스트만 있음
- 외국인이 발음을 모르면 스크립트가 무용지물
- **개선안:** 로마자 발음(Romanization) 병기 또는 TTS(Text-to-Speech) 버튼 추가

---
